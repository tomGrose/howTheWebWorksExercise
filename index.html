Solidify terminology

1. HTTP stands for hyper text transfer protocol. HTTP allows the fetching of resources, and data exchange. It is a transfer protocol for everything from documents, images, videos, etc.

2. URL stands for Uniform Resource Locator. It specifies the location of where data is stored on a server or a computer.

3. DNS stands for Domain Name System. It associates ips to domains on a server.

4. A query string is the part of the URL which assigns values to paramters. It can be data that is being retrieved, or modified data that is being sent to a server.

5. Two different HTTP verbs are post and get. Get allows for the retrieval of information, files, data etc., but does not allow it to be changed. Post can send datat that is meant to be modified.

6. An HTTP request is when a client request something from a server. The server will then respond according to how the request was sent.

7. An HTTP response is what the server sends back to the client after the client sent a request. It will provide the requested material, if it wasn't able to give back anything the matched the request it will reespond with an error code.

8. HTTP headers allow the client and server to pass additional info with a request or response. Request headers: cookie, connection, date Response headers: Content-type, last-modified
